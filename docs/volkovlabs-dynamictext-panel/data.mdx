# Data

You can choose how the retrieved data is passed into the Dynamic Text Panel plugin.

- The `Every row` option applies the **Content** template to every row returned by the query.
- The `All rows` option passes the query results as a single `data` field to the **Content** template.

To work with query results as a whole, you can use [#each builtin helper](../recipes/#iterate-through-all-fields-in-each-record) to iterate through records.

## Every row

Your data source returns a four-column table as follows:

```md
| app  | description                  | cluster | tier     |
| ---- | ---------------------------- | ------- | -------- |
| auth | Handles user authentication. | prod    | frontend |
```

### Template

Apply the following template for each row to display data.

```md
# {{app}}

> {{description}}

{{#if (eq tier "frontend")}}
Link: <a href="https://{{cluster}}.example.com/{{app}}">https://{{cluster}}.example.com/{{app}}</a>
{{/if}}
```

## Render a single template (All rows)

Your data source returns a table with multiple rows as follows: 

| title | author        | year |
| ----- | ------------- | ---- |
| Dune  | Frank Herbert | 1965 |
| 1984  | George Orwell | 1949 |

### Template

With the **All rows** option selected, the template renders a table from the query result as follows:

```
| Title | Author | Year |
| ----- | ------ | ---- |
{{#each data}}
| {{title}} | {{author}} | {{year}} |
{{/each}}
```
