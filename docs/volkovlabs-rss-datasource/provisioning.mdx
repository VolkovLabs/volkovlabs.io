# Provisioning

Grafana supports managing data sources by adding one or more YAML config files in the `provisioning/datasources` folder.

## Example

Example of provisioning the RSS/Atom Data Source.

```yaml
datasources:
  - name: Bitcoin
    type: volkovlabs-rss-datasource
    access: proxy
    orgId: 1
    uid: rZAdZdf7k
    version: 1
    editable: true
    jsonData:
      feed: https://news.bitcoin.com/feed/
```

## Data Sources

When provisioned, the RSS/Atom data source should be available in the data sources list.

### Feed URL

The correct URL must be set

After set the url and test, an error may occur.

<Image
  title="Failed to fetch"
  src="/img/plugins/volkovlabs-rss-datasource/error.png"
/>

If the url is correct and returns data but returns an error, you need to add `/` to the end of the url.

As example use `https://sitename/feed/` insteat `https://sitename/feed`

<Image
  title="Correct URL"
  src="/img/plugins/volkovlabs-rss-datasource/correct-url.png"
/>

:::info Note
But if you already have a data source with `https://sitename/feed` and after add `/` and running the test you will get an error.
After add the `/` at the end, please clear cache and use hard reload
:::
